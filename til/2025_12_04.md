# 2025-12-04 TIL

## 상태를 가지는 컴포저블 작성 규칙

### Compose 이해
- UI를 렌더링할 수 있게 하는 선언형 API

### 선언형 프로그래밍 패러다임
- View를 수동으로 조작하면 휴먼 에러로 에러가 발생할 가능성이 커짐
- Compose는 특정 시점에 UI의 어떤 부분을 다시 그릴지 지능적으로 선택함

### 구성 가능한 함수 (Composable 함수)
- UI로 구성가능한 함수를 컴포저블이라고 부름

### Composition vs Recomposition
- Composition : UI 렌더링
- Recomposition : UI를 다시 렌더링

### 상태 관리의 중요성
- UI는 사용자와의 상호작용에 따라 동적으로 변화해야 한다.
- 상태(state)는 UI의 현재 상황을 나타낸다.

### Composable 의 상태 관리
- 컴포즈는 rememberXXX 형식의 상태 관리용 함수를 제공
- 상태가 변경되면 Compose는 자동으로 UI를 재구성하여 최신 상태를 반영한다

### remember!
- rememberSaveable 을 사용하면 구성 변경시에도 유지된다!
- remember로 만 쓰면 화면 전환 시 초기화 된다
