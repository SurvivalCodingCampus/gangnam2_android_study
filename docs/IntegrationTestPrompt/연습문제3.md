## 연습문제 3: 즐겨찾기 토글 + 상태 유지 테스트

### 🔹1번

```
@app/src/main/java/com/survivalcoding/gangnam2kiandroidstudy/presentation/scre
en/home

즐겨찾기 토글 + 상태 유지 테스트 하고싶어. 
시나리오는 다음과 같아
1. 리스트(LazyRow)에서 북마크 버튼 클릭
2. ViewModel 상태 변경
3. 화면 재구성(recomposition) 후에도 상태 유지


테스트 요구사항도 만족했으면 좋겠어
* 버튼 클릭 후 아이콘 상태 변경
* 화면 회전 or recomposition 상황에서도 상태 유지
```

변경 사항 요약:

1. `DishCard.kt` 수정: 북마크 버튼(Box)에 semantics를 추가했습니다.
   테스트에서 버튼을 식별하기 위한 contentDescription("Bookmark
   Button")과 즐겨찾기 상태를 나타내는 stateDescription("Saved" / "Not
   Saved")을 설정하여 UI 테스트가 가능하도록 개선했습니다.
2. `HomeBookmarkTest.kt` 생성: 홈 화면의 즐겨찾기 토글 기능을 검증하는
   UI 통합 테스트를 추가했습니다.
    * 설정: 테스트용 가짜 저장소(FakeRecipeRepository)와
      HomeViewModel을 주입하도록 Koin 환경을 구성했습니다.
    * 시나리오: HomeRoot를 실행하고 북마크 버튼을 클릭했을 때
      ViewModel의 상태가 변경되어 UI상에서 "Saved"와 "Not Saved"
      상태가 정상적으로 토글되는지 확인합니다.

모든 테스트가 성공적으로 통과되었습니다.
