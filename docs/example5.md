### 통합 테스트 대상이 아닌 것은?

SnackBar 표시 여부
애니메이션 재생 여부
버튼 클릭 시 UseCase 호출 여부
리스트 아이템 개수

### 이유

- SnackBar 표시 여부
  일시적인 UI 피드백이며 비결정적 요소(타이밍, 애니메이션, 메시지 큐)에 의존
  통합 테스트에서는 화면의 지속적인 상태 변화에 집중하는 것이 원칙

- 애니메이션 재생 여부
  애니메이션은 시각적 효과로, 사용자 흐름이나 비즈니스 로직과 무관
  테스트 환경에서도 타이밍·프레임 문제로 불안정

- 버튼 클릭시 UseCase 호출 여부
  이는 ViewModel 내부 로직 검증에 해당
  통합 테스트는 “호출되었는가”가 아니라 “그 결과 UI가 어떻게 변했는가”를 봐야함
  따라서 ViewModel 단위 테스트에서 verify { useCase() } 형태로 검증
